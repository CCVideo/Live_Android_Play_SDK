一、说明

为了用户更加友好和的接入SDK，这里提供一些常见Q&A

如有其他问题，需提供下面信息便于开发人员快速查找并定位问题

二、常见问题

1. 常见错误

1.1 class not found

具体错误如下

1. java.lang.NoClassDefFoundError
2. java.lang.ClassNotFoundException

本地引用如果出现class not found 或者so文件异常无故崩溃的问题，请查询是否完整导入相关类库

1.2 x5混淆错误

如果Debug包运行无错误，打Release包后，报X5内核的运行异常，可以参照X5的混淆配置，进一步调整混淆配置： http://res.imtt.qq.com/TES/proguard.zip

1.3 lib包冲突

如果同时引用本公司点播或者小班课的功能，出现了lib包冲突的问题，需要根据 远程引用冲突问题自查手册 进行排查

1.4 登录直播间失败

解决方案：

  1. 检查权限，参考集成文档
  2. 检查参数，参考集成文档

1.5 Android 9.0 兼容

目前SDK的数据请求绝大部分都使用了Https的方式，但是还是有少数的Http请求。

为更好的适配Android 9.0，建议在AndroidManifest.xml中的Application层添加配置信息：

xxxxxxxxxx
android:networkSecurityConfig="@xml/network_security_config"

然后在res的xml文件夹中添加：network_security_config.xml

xxxxxxxxxx
<?xml version="1.0" encoding="utf-8"?>
<network-security-config>
    <base-config cleartextTrafficPermitted="true" />
</network-security-config>

2.4 登录和播放在同一个界面

部分用户在集成的时候需要将登录和播放在同一个界面，而我们的demo只提供了登录和播放在两个界面的场景范例

如果需要用登录和播放在同一个界面场景，这里只需要关注两个关键点

  1. 方法调用的顺序

    必须在登录的回调方法之后初始化播放器界面和其他模块的界面

xxxxxxxxxx
   // 1. 设置登录参数
   DWLive.getInstance().setDWLiveLoginParams(回调方法, loginInfo)
   // 2. 调用登录接口
   DWLive.getInstance().startLogin();
   
   // 3. 监听回调方法，并在主线程初始化界面
   监听回调方法
       成功--> 开始初始化播放器模块和其他模块的界面
       失败--> 登录失败

  1. 注意demo中LivePlayActivity界面onResume()方法

    登录成功并初始化之后，需要调用开始播放方法，同时需要在onResume方法中设置登录成功的标志，如果登录成功才会调用直播开始的逻辑

xxxxxxxxxx
   //登录成功标志
   private boolean isLoginSuccess;
   
    @Override                                                                                               
    protected void onResume() {
        super.onResume();
        // 判断是否登录成功
         if (isLoginSuccess) {
             ...
             // 调用直播开始的逻辑
             handleLiveStart();
             ...    
         }
    }

4. 升级常见问题

4.1 升级到3.8.0/3.8.1

版本3.8.0首次采用maven集成，如果同时使用本公司点播或者小班课SDK需要注意冲突的问题

3.8.0会出现和点播SDK播放器核心冲突的问题，需使用本地集成或者升级到3.8.1版本

本地集成时请使用播放器版本如下

xxxxxxxxxx
// 集成播放器核心
api('com.bokecc:hdplayer:1.1.0')

该冲突已在3.8.1中已解决

集成时使用直播SDK播放器核心即可

4.2 升级到3.9.0

从版本3.9.0开始为了解决录播时录播文档数据很大导致导致离线回放打开较慢的问题，提出该方案优化打开离线回放的打开速度

升级到3.9.0会出现离线回放旧数据无法播放的问题

解决

升级到3.9.1

demo中UnZiper类将SupZipTool.decompressZipDec替换为SupZipTool.decompressZipDecAndSplitFile方法，参数不变

说明

decompressZipDec方法是对ccr文件的解压，方法依旧可用，但无法进行大文件优化

decompressZipDecAndSplitFile方法是在解压之后进行大数据分离，建议使用该方法

 

5. 用户反馈问题

5.1 直播观看过程中网络断开，重新恢复网络，视频不会自动播放？

说明

在3.9.2之前的版本需要用户自己监听网络变化调用DwLive.restartVideo(Surface surface) 从3.9.2开始sdk底层自动处理

5.2 首次进入不知道直播间的设置？

说明

监听onRoomSettingInfo(SettingInfo info)回调 里面有 数据源类型 是否允许聊天 是否允许问答 房间基础在线人数

5.3 使用回放记忆播放功能？

说明

3.6.0版本之前无法使用记忆播放功能 3.6.0之后可调用该api(DWLiveReplay)

xxxxxxxxxx
   /**
     * 设置上一次播放的位置，可用于记忆播放功能
     */
    public void setLastPosition(long position)

5.4 如何获取随堂测排行榜？

说明

3.9.2之前的调用方式

xxxxxxxxxx
SocketPracticeHandler handler = new SocketPracticeHandler();
handler.getPracticeRanking(viewer listener,practiceId)
   
//并监听回调 onPracticRanking(PracticeRankInfo info)

3.9.2之后调用的方式

xxxxxxxxxx
Dwlive.getInstance().getPracticeRanking(String practiceId)
//并监听回调 onPracticRanking(PracticeRankInfo info)

5.5 System.err: java.io.FileNotFoundException: /storage/emulated/0/bokecc_log/bokecc_sdk.log: open failed: EACCES (Permission denied)。

3.9.1之前版本文件找不到？

说明

该问题分为两种情况 一：先看是否有存储权限 如果没有就去开启权限 二: 是否是运行在android10的设备上

解决方案: 一：没有权限就开启权限 二：如果是android10 设备 如果是 在manifest的application节点添加android:requestLegacyExternalStorage="true"，或者升级到3.9.2 三：sdk初始化的时候 关闭日志采集(不建议)

5.6 视频窗口想做成圆角的或者其他样式？

说明

在视频窗口上添加一个遮罩层

5.7 文档打不开？显示无法打开网页？

说明

检查手机是否设置了代理

5.7 想要直播已经开始时长？

说明

xxxxxxxxxx
Dwlive.getInstance().getLivePlayedTime();   
//通过  onLivePlayedTime回调回去

5.8 想监听播放状态，自定义加载悬浮框？

说明

设置OnInfoListene监听器 首次打开视频显示悬浮框 通过OnInfoListene监听是否开始播放，注意一点音频开播是MEDIA_INFO_AUDIO_RENDERING_START 视频开播是MEDIA_INFO_VIDEO_RENDERING_START 监听到MEDIA_INFO_BUFFERING_START显示加载悬浮框，然后继续监听MEDIA_INFO_AUDIO_RENDERING_START 、MEDIA_INFO_VIDEO_RENDERING_START来隐藏悬浮框

5.9 聊天发送和连麦同时操作造成消息丢失?

说明

服务器接收socket消息有限制，需要延时1秒发送消息

5.10 在线回放无setOnBufferingUpdateListener监听？

说明

在sdk3.9.2之前收不到，3.9.2版本之后才可以收到
